## 7. Notes on Solana Blockchain Auditability

Solana maintains a permanent, immutable transaction ledger. Key implications for this matter:

- **Empty wallets retain full history.** Even when a wallet currently shows a zero balance, every transaction it has ever participated in is permanently recorded. Using `getSignaturesForAddress`, the complete list of transaction signatures can be retrieved, and each transaction can be examined in full detail.
- **Token account closures are recorded.** When a token account is closed on Solana, the closure transaction itself is recorded. The prior balance, the recipient of any remaining tokens, and the timestamp are all preserved.
- **Claims about past holdings are verifiable.** If Travis's report claims that specific wallets held specific token amounts at specific times, these claims can be verified by reconstructing the wallet's transaction history from genesis. If a wallet never received the claimed tokens, no transaction record will exist. If it did receive them, the source, amount, and timing are all visible.
- **This cuts both ways.** Historical verification can confirm or refute claims made by any party, including in this document.
- **Recommended verification scope:** For any legal proceeding, a comprehensive historical audit of the 14 wallets in Travis's report using `getSignaturesForAddress` and `getTransaction` would definitively establish whether these wallets ever held the claimed assets, when they were active, what tokens they transacted, and with whom.
